# Street Fighterメディア「MOC」プロジェクト設計書

## プロジェクト概要

Street Fighterシーンを取り扱う専門メディアプラットフォーム。

### コアコンセプト

- **初心者にもわかりやすい**: 開いた瞬間に「今大会やってる」「こんなにすごい大会なんだ」と理解できる
- **即行動できる**: トップUIは短い情報で即座に行動（観戦）できるようにする
- **観戦熱を高める**: 大会の価値や背景情報を適切に伝え、視聴意欲を喚起する
- **詳細は別ページ**: じっくり読みたい人向けの詳細情報は別ページで提供

### 他メディアとの違い

Street Fighterメディアは、一般的なスポーツメディアとは異なる役割を担う：
- **「今何が配信されているか」の優先度が高い**
- 大会の規模感や参加プレイヤーの豪華さが観戦意欲に直結する
- コミュニティの熱量や「いよいよ感」の演出が重要

## UI/UX設計思想

### ホームページ構造

**週刊発行の「今週の注目一覧」**
- ホーム画面は週刊で発行される注目情報の一覧
- いろいろな要素への**リンク集**であり、一目で何があったか、今週の大会は何かが確認できる
- **ホームはメニューやインデックスではない**: ホーム画面だけで十分に基本的な情報を受け取れる設計
- ホームページだけで完結できる少なすぎず多すぎない情報量
- 要約とURL、画像のみで構成し、ユーザーの時間を取らせない
- 画面遷移で詳しく読む機能はあえて付けない（情報の取捨選択を徹底）

**ナビゲーション設計**
- ホームの上部に遷移可能なメニューを配置
- **グローバルナビゲーションではない**: スクロールに追従しない固定メニューは使用しない
- 紙面を広く使うため、ナビゲーションバーはスクロールと共に上へ消える
- 一般的なWebサイトと同様に、メニューは画面上部に留まらない設計

**新聞スタイルのレイアウト**
- 伝統的な新聞のように、情報が縦にスクロールで展開される
- 各セクションは明確に区切られ、見出しと要約で構成

### 掲載コンテンツ

ホームページに掲載する情報：
- **先週のニュース** (ピックアップ)
- **今週の大会情報** (ピックアップ)
- **最近の界隈** (ピックアップ記事)

すべて要約形式で、その記述だけで満足できるレベルにする。

## 主要機能要件

### 1. 今週の大会情報（週刊サマリー）

**表示要素：**
- 大会の軽い説明（1-2文）
- 配信URL（クリックで即視聴可能）
- 参加者数
- トッププレイヤーの名前
- 大会の「見逃せない度」を示す指標

**具体例：**
- **Capcom Cup**: 「Street Fighter 6の世界最高峰の大会」「賞金総額200万ドル」「世界各地の予選を勝ち抜いた48名」
- **TOPANGA LEAGUE**: 「日本最高峰のリーグ戦」「国内トッププレイヤー16名による総当たり戦」

**設計意図：**
- 「で、これが何なの？」とならないようにする
- 大会の位置付けを知ることで観戦熱を高める
- 大きな大会 = Capcom Cup, EVO, 全国規模の公式大会（プロが参加しない選択肢がないレベル）

### 2. ライブストリーミング表示

**要件：**
- 「今やっている大会」を一目で確認できる
- 大会数は通常1-2個程度なので、すべて表示
- パッと開いて「あ、大会やってるじゃん」→URLから即視聴の流れ

**優先度：**
- Street Fighterでは「今何が配信されているか」の優先度が非常に高い
- 配信中は視覚的に強調（LIVEバッジ、ブランドカラー）

### 3. 大会告知機能

**目的：**
- 大会の「いよいよ感」を演出
- Capcom Cupのような大型大会なら数ヶ月前から告知

**ユーザー体験：**
- 大きな大会を「いよいよ」という心持ちで観戦に臨める
- 前々から話題を共有し、コミュニティの盛り上がりを醸成

## 大会情報の構成要素

### 基本情報（ホームページ掲載）

| 要素 | 説明 | 例 |
|------|------|------|
| 大会名 | 正式名称 | Capcom Cup XI |
| 説明 | 1-2文の簡潔な紹介 | Street Fighter 6の世界最高峰の大会 |
| 開催地 | 大会の規模を示す | Los Angeles, CA / 東京 / オンライン |
| 参加者数 | エントリー数 | 48人 / 256人 |
| 注目選手 | トッププレイヤー名 | ウメハラ、ときど、Punk、MenaRD |
| 配信URL | 即視聴可能なリンク | Twitch/YouTube等 |
| 開催日時 | 日程と時刻 | 2025年11月9日 18:00 - 11月10日 |
| 配信状態 | ライブ/予定 | isLive: true/false |

### オタク情報（詳細ページ）

大会の深い理解と観戦熱を高めるための追加情報：

#### 提供方法の選択肢
1. **大会詳細ページ（独立）**: 独立したページで詳細を提供
2. **ホームからのリンクコーナー**: ホームに専用セクションを設ける
3. **大会カードの「豆知識」ボタン**: 大会カード内にボタンを配置
4. **週刊コラム形式**: 定期的なコラムとして配信

#### 情報の粒度

**レベル1: 名場面**
- 例：「ウメハラのEVO 2004背水の逆転劇」
- その大会を象徴する伝説的な瞬間

**レベル2: 大会の歴史・創設経緯**
- 大会のブランド価値を理解するための背景
- なぜこの大会が権威を持つのか

**レベル3: 会場の特徴・雰囲気**
- 会場の独自性
- 大会特有の文化や伝統

**設計方針：**
- Street Fighter専門wikiとして機能
- ベテランが「この大会はねぇ〜」と語りたくなる情報を集約
- ホームページの要約とは分離し、読みたい人だけが深掘りできる構造

## 情報アーキテクチャ

### 情報の階層化

```
ホームページ（新聞スタイル）
├─ 今週の大会（要約）
│  ├─ 基本情報
│  ├─ 見逃せない度
│  └─ 配信URL
│
├─ 先週のニュース（ピックアップ）
│  ├─ 公式アップデート
│  ├─ プレイヤー動向
│  └─ パッチノート
│
└─ 最近の界隈（ピックアップ記事）
   ├─ コミュニティツール
   ├─ ローカルシーンの動き
   └─ 話題のトピック

詳細ページ（別階層）
├─ 大会詳細ページ
│  ├─ オタク情報
│  ├─ 歴史・名場面
│  └─ 関連記事
│
└─ ニュース詳細ページ
```

### デザイン原則

1. **短い情報で即行動**: トップUIは簡潔に
2. **取捨選択の徹底**: 全部載せない、厳選する
3. **観戦熱の醸成**: 大会の価値を適切に伝える
4. **階層の明確化**: 速報と詳細情報を分離
5. **時間を取らせない**: 要約・URL・画像のみで完結

## ユーザー体験シナリオ

### ペルソナ: 大学3年生のStreet Fighter初心者

**背景:**
- 年齢: 20-21歳
- Street Fighter界隈は初心者
- 課題: "何を見ればいいかわからない"
- 行動: "簡潔に教えてくれる"情報を求める
- デバイス: スマホメイン

### シナリオ1: 初心者の新規ユーザー

1. サイトを開く
2. ヒーローカードで「Capcom Cup XI - Street Fighter 6の世界最高峰、48人、賞金200万ドル」と表示
3. LIVEバッジで「今配信中」と認識
4. 「これは大きな大会だ」と理解
5. 「配信を見る」ボタンをクリックして即視聴

**達成される体験:**
- 「今大会やってる」が即座にわかる
- 「こんなにすごい大会なんだ」と規模感を理解
- 迷わず観戦に移行

### シナリオ2: 熱心なファン

1. サイトを開く
2. 「今週の大会」でCapcom Cup開催を確認
3. 注目選手（ウメハラ、ときど等）を確認
4. 「大会詳細」リンクから詳細ページへ
5. 過去の名場面や歴史を確認
6. 「いよいよ感」を持って観戦へ

**達成される体験:**
- 大会の歴史的価値を理解
- コミュニティの盛り上がりを共有
- より深い観戦体験

### シナリオ3: 大会告知期間

1. 3ヶ月前にCapcom Cup告知が掲載される
2. 週次でカウントダウンや関連情報が更新
3. コミュニティで「いよいよだね」という機運が高まる
4. 開催当日、準備万端の状態で観戦

**達成される体験:**
- 「いよいよ感」の醸成
- 前々からの話題共有
- 観戦に向けた期待の高まり

## 今後の検討事項

1. **大会の「見逃せない度」の評価基準**: 定量的な指標の設計
